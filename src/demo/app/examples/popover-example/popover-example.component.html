By default dropdown panel is rendered in the normal document flow, not in the browser's native top layer.

If ng-select is placed inside an element with overflow: hidden, or inside a component that is already rendered in a top-layer, the dropdown panel can be clipped or appear behind that container.

It can be fixed by setting [popover]="true", which forces the dropdown panel to be displayed in a top-layer.

Without popover - dropdown stays in normal flow

With popover - dropdown is rendered in a top-layer

Use [popover]="true" when:
  • The select is inside a container with overflow: hidden, overflow: auto or overflow: clip
  • The select is rendered inside a modal, dialog or any other component already using the HTML top-layer
  • The dropdown appears behind another element because of stacking context or z-index constraints

Avoid it when:
  • You are already using appendTo; both options solve similar positioning problems and should not be combined
  • You need to support browsers without the Popover API (Chrome 114+, Firefox 125+, Safari 17+)
