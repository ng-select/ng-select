Ng-Select supports the following accessibility in mind:

  • Dropdown Keyboard Interaction:
    • Enter: If the menu is Closed will Open it (If openOnEnter isn't false), also to choose an elmenet from the dropdown.
    • ESC: Closes the menu if it is opened.
    • Space: Opens the menu if it is closed first time use, after that just moving the key couror.
    • BackSpace: Clears the menu items one by one if it is multiple, or just clear the existing item if it is not multiple
    • Key Up: Moving inside the menu from bottom to top, also continue cycling from bottom to top
    • Key down: Moving inside the menu from top to bottom, also continue cycling from top to bottom
  • Menu items Keyboard Interaction:
    • Clear item X button is keyboard access:
    • Enter or Space: Removes the item then sets the focus back to the ng-select input
  • Clear all items Keyboard Interaction:
    • Clear all item X button is keyboard access:
    • Enter or Space: Removes all the items then sets the focus back to the ng-select input

Labeling for Screen Readers:

  • Input Field:
    • If there is no items, the place holder will be announced if it is exists.
    • If There are multible items , also the place holder will be announced if it is exists.
    • If there is only one item will be announced.
    • If there is a custom label, the label will be announced first then the input field
  • Dropdown:
    • If it is a checkbox will be announced the visible label of it and it is states.
    • If it is not a checkbox will be announced the visible label of it.
  • Multi items with clear x button:
    • If you did not provide a custom [removeText], when the focus gets on the remove button, will be announced Remove then the item visible title
  • Clear all text x button:
    • If you did not provide a custom [clearAllText] word, when the focus gets on the clearAllText button, will be announced Clear all

Alerting for Screen Readers:

  • If there is No Result Found.
  • In Case of type To Search Text.
  • If there is loading Text.
